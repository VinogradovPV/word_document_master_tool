# Excel PDF export

## Что поддерживается

MVP поддерживает Excel-источники `.xls` и `.xlsx`. Временные Office-файлы вида
`~$*.xls` и `~$*.xlsx` игнорируются при обновлении списка документов.

## Требования

Экспорт Excel в PDF работает через Microsoft Excel COM automation, поэтому для
реального выполнения нужны Windows, установленный Microsoft Excel и `pywin32`.
Обычные unit-тесты используют mock-объекты и не требуют Microsoft Excel.

## Как включить

1. Откройте GUI командой `wdmt-gui`.
2. На вкладке `Главное` выберите исходную папку и папку результата.
3. Нажмите `Обновить список`.
4. На вкладке `PDF` включите `Экспортировать XLS/XLSX в PDF`.
5. Укажите папку PDF.
6. Нажмите `Выполнить выбранные операции`.

## Поведение MVP

- Экспортируется вся книга.
- Книга открывается read-only.
- Исходный Excel-файл не сохраняется и не изменяется.
- Параметры Word-обработки, нумерации, сносок и исправлений к Excel не применяются.
- В TSV-лог для Excel пишутся `-` в полях страниц, сносок и надстрочных символов.
- При успехе сообщение лога: `Excel exported to PDF`.

## Общий PDF

Если включено `Создать общий PDF из созданных PDF`, общий PDF собирается через
`pypdf` из `item.pdf_path` в порядке таблицы документов. Сервис слияния не
различает Word и Excel, поэтому в общий PDF попадают все уже созданные PDF-файлы.

## Ограничения

- `.xlsm` пока не включен в discovery.
- Настройки области печати не вынесены в GUI: MVP экспортирует всю книгу с учетом
  существующих областей печати Excel.
- Реальные COM-сценарии требуют ручной проверки на машине с Microsoft Excel.
