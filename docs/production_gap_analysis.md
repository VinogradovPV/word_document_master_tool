# Production gap analysis

## Текущее состояние

Текущий Python GUI запускается как компактное Tkinter-приложение с выбором исходной папки, папки результата, таблицей документов, базовыми PDF-чекбоксами, progressbar и кнопками обработки, слияния и разделения. По сравнению с VBA-референсом `Word Document Master Tool v1.1` интерфейс пока покрывает только каркас основных сценариев и не представляет большую часть настроек VBA-формы.

Backend-слой уже содержит модели для многих настроек, сервисы Word/PDF и TSV-логирование, но значительная часть этих возможностей не выведена в GUI или требует интеграционной проверки с Microsoft Word.

## Что реализовано

- Выбор исходной папки.
- Выбор папки результата.
- Обзор папок через стандартный диалог.
- Обновление списка документов.
- Поиск поддерживаемых Word-документов `.docx`, `.docm`, `.doc`, `.rtf`.
- Таблица документов с колонками `Файл` и `Статус`.
- Счетчики `Найдено` и `Выбрано`.
- Перемещение выбранного документа вверх и вниз.
- Переключение выбора документа двойным кликом.
- Базовые чекбоксы PDF исходников и PDF результата.
- Выбор папки PDF.
- Кнопки обработки, слияния и разделения.
- TSV-лог с колонкой `Надстрочные символы, стр.`.
- PDF merge через `pypdf`.
- Smoke-импорт GUI.

## Что реализовано частично

- Блок `Папки и результат`: базовые controls представлены; требуется проверка полного сценария сохранения итогового имени.
- Блок `Документы`: основные колонки и кнопки представлены; `Проверить` пока выполняет только легкую GUI-проверку без Word COM.
- Блок `PDF-экспорт`: часть настроек есть в модели и сервисах, но не выведена в GUI.
- Блок `Статус и действия`: progressbar есть, но нет отдельной строки статуса, просмотра лога, кнопок сброса и закрытия.
- Word/PDF backend требует интеграционной проверки на Windows с Microsoft Word.
- Валидация конфликтующих настроек есть для части правил, но GUI пока не использует все соответствующие controls.

## Что отсутствует

- Полный GUI-блок настроек объединения.
- GUI-блок исправлений и комментариев.
- GUI-блок нумерации страниц.
- GUI-блок маркеров и удаления маркеров.
- Полный GUI-блок разделения по маркерам.
- GUI-блок сносок и концевых сносок.
- Полная карта статусов backend/stub для каждой кнопки.
- Пользовательские документы `docs/user_manual.md`, `docs/local_setup.md`, `docs/testing.md`, `docs/troubleshooting.md`, `docs/production_readiness.md`.

## Что отложено

- Полная Word COM интеграция в обычных unit-тестах: такие проверки должны оставаться `windows_word` / integration.
- Pixel-perfect копирование VBA-формы: проект должен сохранять Python/Tkinter-архитектуру.
- Массовая переработка `main_window.py`: вынос блоков в `gui/widgets/` нужен по мере роста GUI.
- Полная production-ready сертификация до прохождения CI, документации и ручной проверки Word-сценариев.

## Риски

- GUI не соответствует VBA v1.1.
- Часть сервисов может быть stub или не покрыта интеграционными тестами.
- Word COM требует Windows и установленный Microsoft Word.
- PDF merge должен работать только через `pypdf`.
- Исходники нельзя пересохранять.
- `PDF исходников без изменений` не должен применять обработку, нумерацию, сноски, поля или обновление полей.
- Stub-функции нельзя считать production-ready.
