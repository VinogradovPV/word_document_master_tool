# Production gap analysis

## Текущее состояние

Python GUI реорганизован на `ttk.Notebook`, чтобы форма помещалась в окно и нижняя панель действий всегда оставалась видимой. Основные VBA-блоки разнесены по вкладкам `Главное`, `Документы`, `Обработка`, `Нумерация страниц`, `PDF`, `Маркеры`, `Сноски`, `Журнал`.

Backend-слой уже содержит модели для большинства настроек, сервисы Word/PDF и TSV-логирование. Текущий этап не добавляет новую Word/PDF backend-логику: часть кнопок и сценариев по-прежнему требует интеграционной проверки с Microsoft Word.

## Что реализовано

- Вкладочный GUI на `ttk.Notebook`.
- Всегда видимая нижняя панель действий со статусом, progressbar, кнопками `Обработать файлы`, `Объединить`, `Сбросить`, `Закрыть`.
- Вкладка `Главное` с папками, именем результата и сводкой.
- Вкладка `Документы` с колонками `Вкл`, `Файл`, `Тип`, `Размер`, `Изменён`, `Статус`.
- Кнопки документов: `Проверить`, `Вверх`, `Вниз`, `Вкл`, `Выкл`, `Выбрать все`, `Снять все`.
- Двойной клик по колонке `Вкл` переключает `DocumentItem.is_selected`.
- Вкладка `Обработка` с настройками объединения и исправлений/комментариев.
- Вкладка `Нумерация страниц` с взаимоисключающими режимами нумерации и колонтитулов.
- Вкладка `PDF` с PDF-режимами, папкой PDF, качеством `Печать`/`Экран`, PDF/A и свойствами.
- Вкладка `Маркеры` с настройками маркеров, кнопкой разделения и stub для удаления маркеров.
- Вкладка `Сноски` с режимами нумерации сносок и концевых сносок.
- Вкладка `Журнал` с текстовым журналом, кнопками открытия log-файла/папки, очистки и копирования диагностики.
- Dev extras заданы через pip-compatible `[project.optional-dependencies]`.
- GitHub Actions устанавливает `.[dev]` и запускает `python -m ruff`.
- Smoke-импорт GUI.

## Что реализовано частично

- `Проверить` выполняет легкую GUI-проверку статусов без Word COM.
- `Обработать файлы`, `Объединить` и `Разделить` зависят от Word COM и требуют ручной интеграционной проверки.
- `Удалить маркеры` пока честный backend-stub.
- PDF-режимы наименования и `Открыть PDF` имеют GUI-настройки, но требуют backend-проверки.
- Форматы нумерации страниц и сносок требуют проверки полноты применения в Word-сервисах.
- Открытие log-файла/папки зависит от появления реального файла в ходе обработки.

## Что отсутствует

- Пользовательские документы `docs/user_manual.md`, `docs/local_setup.md`, `docs/testing.md`, `docs/troubleshooting.md`, `docs/production_readiness.md`.
- Полная ручная проверка `wdmt-gui` с Microsoft Word и реальными документами.
- Интеграционные тесты `windows_word` для Word COM сценариев.

## Подключенные backend-сервисы

- `DocumentDiscovery` для обновления списка документов.
- `SourceProcessingService` для обработки выбранных файлов.
- `WordMergeService` для объединения документов.
- `WordSplitService` для разделения по маркерам.
- `PdfExportService` для PDF export.
- `PdfMergeService` для объединения созданных PDF через `pypdf`.
- `ProcessingLogger` и `setup_application_logging` для TSV/application logs.

## Stub-кнопки

- `Удалить маркеры`: показывает сообщение `Функция пока не подключена к backend.`

## Риски

- Word COM требует Windows и установленный Microsoft Word.
- Исходные документы нельзя пересохранять.
- `PDF исходников без изменений` не должен применять исправления, комментарии, Track Changes, нумерацию страниц, сноски, поля или обновление полей.
- PDF merge должен оставаться только через `pypdf`.
- Stub-функции нельзя считать production-ready.
- CI нужно подтвердить на GitHub после push.
