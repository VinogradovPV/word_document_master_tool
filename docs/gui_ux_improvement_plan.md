# План UX-доработки GUI

## Что изменено

GUI переведен из набора технических вкладок в workflow-структуру:

- `Главное`
- `Документы`
- `Word`
- `Нумерация`
- `PDF`
- `Разделение`
- `Журнал`

Нижняя панель действий остаётся вне вкладок и всегда видима.

## Почему объединены нумерация страниц и сноски

Нумерация страниц, обычных сносок и концевых сносок часто настраивается в одном рабочем сценарии. Отдельные вкладки `Нумерация страниц` и `Сноски` создавали пустое пространство и заставляли пользователя прыгать между связанными настройками. Теперь они объединены во вкладку `Нумерация`, где рядом видны настройки страниц, сносок и предпросмотр.

## Глобальные кнопки

Нижние кнопки являются глобальными:

- `Выполнить выбранные операции`
- `Объединить Word-документы`
- `Сбросить`
- `Закрыть`

Они применяют настройки со всех вкладок. Над кнопками добавлено пояснение: `Глобальные действия: применяют настройки со всех вкладок.`

## Настройки со всех вкладок

Перед запуском операции GUI собирает `ToolSettings` из вкладок:

- папки и имя результата из `Главное`;
- выбранные документы и порядок из `Документы`;
- настройки Word-объединения и обработки из `Word`;
- нумерацию страниц и сносок из `Нумерация`;
- PDF-настройки из `PDF`;
- маркеры и split-настройки из `Разделение`.

## План выполнения

На вкладке `Главное` добавлен блок `План выполнения`. Он показывает:

- сколько документов выбрано;
- будет ли создан итоговый Word;
- будут ли созданы обработанные копии;
- какие PDF будут созданы;
- будет ли создан общий PDF;
- предупреждения;
- ошибки.

План можно скопировать из вкладки `Журнал` кнопкой `Копировать план выполнения`.

## Stub-операции

На текущем этапе без новой backend-логики остаётся stub:

- `Удалить маркеры`: показывает `Функция пока не подключена к backend.`

## Добавленные предупреждения

- PDF исходников без изменений игнорирует исправления, нумерацию и сноски.
- PDF обработанных копий требует создания обработанных копий.
- Общий PDF создаётся только из уже созданных отдельных PDF через `pypdf`.
- Полное удаление маркеров может изменить документы.

## Следующий этап

Следующий MVP-фокус:

1. PDF исходников без изменений.
2. Общий PDF из созданных PDF через `pypdf`.
3. TSV-лог и открытие папки результата.
4. Открытие log-файла.
5. Только потом Word merge, обработанные копии, нумерация, сноски, маркеры и split.
