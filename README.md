# Word Document Master Tool

Python-инструмент для пакетной обработки документов Microsoft Word, их слияния, нумерации и экспорта в PDF.

## Основные возможности

Инструмент предоставляет графический интерфейс для автоматизации рутинных задач при работе с большим количеством документов:

- **Пакетная обработка**: Принятие исправлений, удаление комментариев и отключение режима отслеживания изменений во всех выбранных файлах.
- **Умное слияние**: Объединение документов в один файл с сохранением структуры и возможностью настройки разрывов.
- **Профессиональная нумерация**: Добавление номеров страниц в колонтитулы и управление нумерацией сносок (сквозная или по разделам).
- **Экспорт в PDF**: Создание PDF-версий как для отдельных файлов, так и для объединенного результата.
- **Контроль качества**: Автоматический поиск надстрочных знаков и формирование детального TSV-отчета для проверки страниц.

## Установка

Для работы требуется Windows и установленный Microsoft Word.

```bash
# Клонирование репозитория
git clone https://github.com/VinogradovPV/word_document_master_tool.git
cd word_document_master_tool

# Установка зависимостей
pip install .
```

## Использование

### Графический интерфейс
Запустите приложение командой:
```bash
wdmt-gui
```
Или через Python:
```bash
python -m word_document_master_tool.gui.app
```

### Консольный интерфейс
Для проверки версии или получения справки:
```bash
wdmt --version
wdmt --help
```

## Структура проекта

| Папка | Описание |
|-------|----------|
| `src/core` | Модели данных, логирование и базовая логика. |
| `src/gui` | Код интерфейса и кастомные виджеты. |
| `src/word` | Сервисы автоматизации Microsoft Word. |
| `src/pdf` | Сервисы экспорта и слияния PDF. |
| `docs` | Техническая документация и отчеты об аудите. |

## Разработка

Проект использует `ruff` для линтинга и `pytest` для тестирования. 
Для запуска тестов в Linux (без Word) используйте:
```bash
pytest -m "not windows_word"
```
