# Word Document Master Tool (Python Edition)

Python-инструмент для пакетной обработки документов Word и экспорта в PDF. Аналог VBA-макроса Word Document Master Tool v1.1.

## Основные возможности

*   **Поиск документов**: Автоматическое обнаружение файлов `.docx`, `.docm`, `.doc`, `.rtf` в выбранной папке.
*   **Слияние документов**: Объединение нескольких файлов в один с поддержкой различных типов разрывов (новая страница, новый раздел и т.д.).
*   **Обработка копий**: Автоматическое принятие правок, отключение режима исправлений и удаление комментариев в копиях документов.
*   **Нумерация**: Гибкая настройка нумерации страниц и сносок (сквозная или сброс для каждого документа).
*   **Экспорт в PDF**: Сохранение исходников, обработанных копий или итогового документа в формате PDF.
*   **Сканер надстрочных знаков**: Автоматический поиск страниц, содержащих надстрочные знаки (superscript), для формирования отчета о необходимости ручной проверки.
*   **Маркеры и разделение**: Вставка технических маркеров для последующего автоматического разделения объединенного документа.

## Системные требования

*   **ОС**: Windows (для работы через Word COM-автоматизацию).
*   **ПО**: Microsoft Word (установленный в системе).
*   **Python**: 3.12 или выше.

## Установка

1.  Клонируйте репозиторий.
2.  Установите зависимости:
    ```bash
    pip install .
    ```

## Запуск

### Графический интерфейс (GUI)
```bash
wdmt-gui
```

### Командная строка (CLI)
```bash
wdmt --help
```

## Инструкция пользователя

1.  **Выберите папки**: Укажите путь к исходным файлам и папку для сохранения результатов.
2.  **Настройте список**: Используйте кнопки "Вверх"/"Вниз" для изменения порядка файлов. Флажки в колонке "Вкл" определяют, какие файлы будут обработаны.
3.  **Выберите параметры**:
    *   Включите нумерацию страниц или сносок, если требуется.
    *   Настройте экспорт в PDF.
    *   Выберите режим слияния.
4.  **Запустите процесс**:
    *   "Обработать файлы" — создаст обработанные копии каждого файла.
    *   "Слияние документов" — создаст один общий файл.
5.  **Проверьте лог**: В папке результата будет создан TSV-файл с подробным отчетом об обработке.

## Разработка

Проект использует `ruff` для линтинга и `pytest` для тестирования.

```bash
# Проверка кода
ruff check .

# Запуск тестов
pytest
```

### Python Version Fallback
The project was initialized on a system with Python 3.12.3. As Python 3.14 was not available, the `pyproject.toml` has been adjusted to target Python 3.12:
- `requires-python = ">=3.12,<3.15"`
- `target-version = "py312"` (for Ruff)
